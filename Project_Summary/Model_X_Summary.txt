PIVOT IDENTIFIKATION (nur auf 3D, W, M)
================================================================================

⚠️ IDENTISCH ZU MODEL 3

BULLISH PIVOT:
- Kerze 1: Rot (bearish, close < open)
- Kerze 2: Grün (bullish, close > open)
- ⚠️ Pivot existiert erst NACH Close von Kerze 2

BEARISH PIVOT:
- Kerze 1: Grün (bullish, close > open)
- Kerze 2: Rot (bearish, close < open)
- ⚠️ Pivot existiert erst NACH Close von Kerze 2

DOJI-FILTER:
- Wenn Body < 5% der gesamten Kerze (High - Low) → Pivot nicht generieren
- Gilt für BEIDE Kerzen des Pivots
- Standard: 5% (testbar: 3%, 7%, kein Filter)

VERSATZ-HANDLING (Close K1 ≠ Open K2):
Wenn Close K1 und Open K2 nicht den gleichen Preis haben:

1. Berechne beide Varianten:
   - Variante A: Close Kerze 1 bis Pivot Extreme
   - Variante B: Open Kerze 2 bis Pivot Extreme

2. Standard-Regel: Nimm die Variante mit größerer Pip-Größe als Pivot

3. Versatz-Filter: 
   - Wenn größere Variante ≥ 2x kleinere Variante → Pivot ungültig
   - Standard: 2x Filter aktiv

Beispiel Bullish:
- Pivot Extreme: 95€
- Open K2: 110€ → Gap: 15€
- Close K1: 120€ → Gap: 25€
- Pivot wird bei 120€ gesetzt (Close K1, größere Box)
- Wenn 25€ ≥ 2 × 15€ (=30€) → würde Pivot gelöscht (nicht der Fall hier)

ZU TESTENDE VERSATZ-VARIANTEN:
- Größere Variante nehmen (Standard)
- Kleinere Variante nehmen
- Immer Close K1 nehmen
- Immer Open K2 nehmen
- Versatz-Filter bei 1.5x aktivieren
- Versatz-Filter bei 2x aktivieren (Standard)
- Kein Versatz-Filter

PIVOT-STRUKTUR:

Terminologie:
- Pivot:           Close K1 ODER Open K2 (größere Box-Variante)
- Pivot Extreme:   Ende der längeren Wick
                   (bullish: tiefster Low, bearish: höchster High)
- Pivot Near:      Ende der kürzeren Wick
                   (bullish: höherer Low, bearish: tieferer High)
- Pivot Gap:       Box von Pivot bis Pivot Extreme
- Wick Difference: Box von Pivot Near bis Pivot Extreme

FIBONACCI-SETUP:

BULLISH PIVOT:
- Fib 0   = Pivot (oben)
- Fib 1   = Pivot Extreme (unten)
- Fib 1.5 = SL (unter Pivot Extreme)
- Fib -3  = TP (über Pivot)

BEARISH PIVOT:
- Fib 0   = Pivot (unten)
- Fib 1   = Pivot Extreme (oben)
- Fib 1.5 = SL (über Pivot Extreme)
- Fib -3  = TP (unter Pivot)

WIE FIBONACCI BERECHNEN:
1. Setze Fib 0 auf Pivot-Level
2. Setze Fib 1 auf Pivot Extreme
3. Fibonacci-Tool zieht automatisch weitere Levels

BERECHNUNG DER FIB-LEVELS:
Die Fib-Levels basieren auf der Distanz zwischen Fib 0 und Fib 1:

Pivot Gap Größe = |Pivot - Pivot Extreme|

Bullish:
- Fib 1.5 = Pivot Extreme - (1.5 × Pivot Gap Größe)
- Fib -3  = Pivot + (3.0 × Pivot Gap Größe)

Bearish:
- Fib 1.5 = Pivot Extreme + (1.5 × Pivot Gap Größe)
- Fib -3  = Pivot - (3.0 × Pivot Gap Größe)

BEISPIEL BULLISH:
- Pivot: 1.2000
- Pivot Extreme: 1.1900 (100 Pips unter Pivot)
- Pivot Gap Größe: 100 Pips

Fib Levels:
- Fib 0:   1.2000 (Pivot)
- Fib 1:   1.1900 (Pivot Extreme)
- Fib 1.5: 1.1900 - (1.5 × 100) = 1.1750 (SL)
- Fib -3:  1.2000 + (3.0 × 100) = 1.2300 (TP)

Entry bei: 1.1950 (in der Gap)
SL:        1.1750 (200 Pips unter Entry)
TP:        1.2300 (350 Pips über Entry)
RR:        350 / 200 = 1.75 RR → Wird auf 2:1 angepasst

RISK/REWARD MANAGEMENT
================================================================================

FIXE 2:1 RR (Standard):

✓ IMMER EXAKT 2:1 RISK/REWARD
- Keine Anpassungen
- Keine Mindest- oder Maximal-Limits
- Durch Fib 1.5 (SL) und Fib -3 (TP) automatisch gegeben

BERECHNUNG (zur Verifikation):

Bullish (Long):  RR = (TP - Entry) / (Entry - SL)
Bearish (Short): RR = (Entry - TP) / (SL - Entry)

WARUM 2:1 RR?

Mathematik:
- Pivot Gap = Distanz von Fib 0 zu Fib 1 = 1.0 Einheit
- SL bei Fib 1.5 = 1.5 Einheiten vom Fib 0
- TP bei Fib -3 = 3.0 Einheiten vom Fib 0

Bei Entry in der Mitte der Gap (Worst Case für RR):
- Entry bei 0.5 (Mitte zwischen 0 und 1)
- Distanz zu SL: 0.5 + 1.5 = 2.0 Einheiten
- Distanz zu TP: 3.0 + 0.5 = 3.5 Einheiten
- RR: 3.5 / 2.0 = 1.75 RR

Bei Entry an Pivot (Best Case):
- Entry bei 0.0 (Fib 0)
- Distanz zu SL: 1.5 Einheiten
- Distanz zu TP: 3.0 Einheiten
- RR: 3.0 / 1.5 = 2.0 RR

Bei Entry an Pivot Extreme (Best Case bearish/bullish umgekehrt):
- Entry bei 1.0 (Fib 1)
- Distanz zu SL: 0.5 Einheiten
- Distanz zu TP: 4.0 Einheiten
- RR: 4.0 / 0.5 = 8.0 RR

DURCHSCHNITTLICH: ~2:1 RR (abhängig von Entry-Punkt in Gap)

UNTERSCHIED ZU MODEL 3:

Model 3:
- Variable RR zwischen 1.0 und 1.5
- Mit Anpassungen wenn außerhalb
- Komplexes RR Management

Model X:
- Fixe RR von ~2:1
- Keine Anpassungen
- Simples Set & Forget

KEIN RR-CHECK VOR ENTRY:
⚠️ Im Gegensatz zu Model 3 gibt es KEINEN RR-Check vor Entry
- Jedes Setup wird getradet
- Egal welches RR sich ergibt
- Vereinfachung des Systems
