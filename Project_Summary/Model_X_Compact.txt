# Model X - Kompakte Strategie-Übersicht

## Was ist Model X?
Vereinfachte Pivot-Gap-Trading-Strategie für Forex - OHNE Verfeinerungen, direkter Entry, fixe SL/TP.

## Pivot-Erkennung (nur 3D, W, M)
- **2-Kerzen-Muster:**
  - Bullish: K1 rot (close < open), K2 grün (close > open)
  - Bearish: K1 grün, K2 rot
  - Pivot gilt erst NACH Close von K2

- **Filter:**
  - Doji-Filter: Body ≥ 5% der Kerze (sonst verwerfen)
  - Versatz-Filter: Wenn Close K1 ≠ Open K2 → größere Box wählen; wenn größere ≥ 2× kleinere → Pivot ungültig

## Pivot-Bauteile
- **Pivot (Fib 0):** Close K1 oder Open K2 (größere Box)
- **Pivot Extreme (Fib 1):** Ende der längeren Wick (bullish = tiefster Low, bearish = höchster High)
- **Pivot Gap:** Box zwischen Pivot und Pivot Extreme = Entry-Zone
- **SL (Fib 1.5):** 1.5× Gap-Größe jenseits Pivot Extreme
- **TP (Fib -3):** 3.0× Gap-Größe jenseits Pivot

## Entry-Logik (Standard)
- Entry bei Touch der Pivot Gap (direkter Market Order, kein Close abwarten)
- Varianten testbar: 1H/4H Close, engere Entry-Zonen (Wick Diff, Mitte Gap)

## Risk/Reward
- Fixe ~2:1 RR (durch Fib 1.5 / -3 automatisch)
- Keine Anpassungen, Set & Forget

## Daten
- Quelle: Oanda API (UTC)
- Timeframes: 3D, W, M (für Pivots), H1 (für Entry-Prüfung falls nötig)
- Format: CSV/Parquet, timezone-aware UTC
- Wichtig: Zeitstempel in UTC speichern, beim Vergleich mit TradingView in Europe/Berlin konvertieren

## Backtest-Ablauf
1. Pivots auf 3D/W/M erkennen (2-Kerzen-Muster + Filter)
2. Entry bei Pivot Gap Touch (direkter Touch oder Close-Variante)
3. Trade bis SL (Fib 1.5) oder TP (Fib -3)
4. Dokumentiere Wins/Losses, berechne Performance-Metriken

## Nächste Schritte
- Pivot-Export für W/M/3D (nur Pivot-Daten: Datum UTC+Berlin, Gap High/Low, OHLC)
- Backtest für Standard-Setup (Pivot Gap Touch, direkter Entry)
- Performance-Auswertung (Win-Rate, RR, Drawdown)
- Optional: Test von Entry/Exit-Varianten

## Kritische Punkte
- Pivots nur auf 3D/W/M (nicht auf kleineren Timeframes)
- UTC-Zeitstempel konsistent nutzen (Abgleich mit TradingView über Europe/Berlin)
- Keine Verfeinerungen (Hauptunterschied zu Model 3)
- Strategie basiert stark auf Fundamentals (COT, Seasonality, Valuation) - ohne diese wahrscheinlich nicht profitabel
